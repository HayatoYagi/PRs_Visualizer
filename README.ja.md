# GitHub PRs Visualizer

[🇬🇧 English](./README.md) | 🇯🇵 日本語

GitHubリポジトリのオープンなプルリクエストを視覚的に把握するためのデスクトップアプリケーションです。WinDirStatやWizTreeのようなディスク容量可視化ツールにインスパイアされたツリーマップ形式で、どのファイルやディレクトリで開発が進行中かを一目で確認できます。

## 🎯 このツールを使って欲しい人

### チーム開発をしている方
- 複数人で同時に開発していて、**コンフリクトが起きそうな場所を事前に把握**したい
- PRレビューの優先順位を決めたい
- コードベースのどこで活発に開発が行われているかを把握したい

### AI Agentを活用している方
- AIエージェントに並列でPRを作らせている時に、**作業範囲を監視**したい
- 大量のPRに対して影響範囲を視覚的に確認し、レビュー優先度を決めたい

### アーキテクチャの改善を検討している方
- 一箇所で頻繁にコンフリクトが起こっている場合、**アーキテクチャの見直しのきっかけ**を得られる
- 適切にモジュール化・ファイル分割が行われているかを視覚的に確認したい

## ✨ 主な機能

### 📊 Treemapビジュアライゼーション
- **ファイルとディレクトリを面積で表現**: ファイルサイズ（行数）に比例した矩形で表示
- **階層的な表示**: ディレクトリ構造をネストで表現
- **ズーム機能**: マウスホイールで拡大・縮小し、深い階層まで探索可能

### 🎨 変更内容の可視化
- **色分けで変更の種類を表示**:
  - 🟢 緑: 追加 (Addition)
  - 🟡 黄: 編集 (Modification)
  - 🔴 赤: 削除 (Deletion)
- **PR毎に異なる枠線色**: 各PRまたは作成者に固有の色を割り当て
- **変更密度の表現**: 変更行数の割合に応じて色の濃さが変化
- **コンフリクトの警告**: 複数のPRが同じファイルを変更している場合にハイライト表示

### 📋 PR一覧サイドバー
- オープンなPR全てを一覧表示
- Draft PRの表示/非表示切り替え
- 各PRの表示/非表示を個別に切り替え可能
- PRアイテムをクリックしてGitHub上のPRページをブラウザで開く

## 🚀 使い方

### 前提条件
- Java 17以上がインストールされていること

### 起動方法

#### macOS / Linux
```bash
./gradlew :composeApp:run
```

#### Windows
```bash
.\gradlew.bat :composeApp:run
```


## 🔧 技術スタック
