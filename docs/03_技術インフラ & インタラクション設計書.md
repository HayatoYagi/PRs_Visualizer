# **GitHub PRs Visualizer: 技術インフラ & インタラクション設計書**

## **1\. データインフラ設計 (GitHub API 連携)**

効率的なデータ取得とレートリミット対策、およびオフライン体験の向上のための戦略。

### **1.1. GraphQL API の採用**

REST APIでは複数回のリクエストが必要な情報を、GraphQLで1回に集約する。

* **取得項目**:  
  * リポジトリの defaultBranch の tree (recursive)  
  * Open状態の pullRequests 一覧  
  * 各PRの files (パス、additions、deletions)  
  * 各ファイルの最終更新日（オプション、色の強弱に利用）

### **1.2. キャッシュ戦略**

リポジトリのファイル構造は頻繁に変化しないため、ローカルキャッシュを活用する。

* **Tree Cache**: ファイルツリー構造をローカル（SQLDelight または JSONファイル）に保存。  
* **Diff Cache**: PRごとの変更ファイル情報を保存。  
* **更新トリガー**:  
  * アプリ起動時、または手動更新ボタン押下時。  
  * lastCommitHash をチェックし、変更がある場合のみ再取得。

### **1.3. エラーハンドリングとレートリミット**

* **Exponential Backoff**: API呼び出し失敗時に、1s, 2s, 4s... と間隔を空けて最大5回リトライ。  
* **Rate Limit Monitor**: 残りのリクエスト可能回数をUIに表示。

### **1.4. 認証戦略 (Authentication)**

シームレスな体験のため、以下のフローをメインに採用する。

1. **OAuth 2.0 Authorization Code Flow (Local Loopback) \- 推奨**:  
   * **ユーザー体験**:  
     1. アプリ内の「Login with GitHub」ボタンをクリック。  
     2. 標準ブラウザでGitHubの承認ページが自動で開く。  
     3. ユーザーが「Authorize」をクリック。  
     4. ブラウザに「認証完了」と表示され、アプリが自動的にログイン状態になる。  
   * **技術的実装**:  
     1. アプリ側で一時的なローカルHTTPサーバー（http://localhost:PORT）を起動。  
     2. java.awt.Desktop を使用して、client\_id と redirect\_uri を含む認可URLをブラウザで開く。  
     3. GitHubからのリダイレクト（認可コードを含む）をローカルサーバーで受け取る。  
     4. 受け取ったコードをクライアントシークレット（またはPKCE）を用いてアクセストークンに交換。  
     5. サーバーをシャットダウン。  
2. **Personal Access Token (PAT) \- 予備**:  
   * ネットワーク環境等の理由でブラウザ連携ができない場合のエスケープハッチとして、手動入力UIも提供。

## **2\. インタラクション設計**

WinDirStat/WizTreeのような軽快な操作性をCompose for Desktopで実現する。

### **2.1. ズーム & パン (Navigation)**

* **スクロールホイール**: カーソル位置を中心にズームイン/アウト。  
* **ドラッグ (右クリック or 中クリック)**: 表示領域のパン（移動）。  
* **ダブルクリック**:  
  * **ディレクトリ**: そのディレクトリをルートとする「Deep Diveモード」へ移行。  
  * **ファイル**: ブラウザでそのファイルを開く。

### **2.2. ホバー & セレクション (Inspection)**

* **マウスホバー**:  
  * 矩形のハイライト（枠線を太く、または発光）。  
  * ツールチップ表示：ファイル名、総行数、関与しているPR番号・Author名。  
* **クリック**:  
  * サイドバーのPRリストと連動。そのファイルに関係するPRをリスト内で強調。

### **2.3. PRフィルタリング (Filtering)**

* **Sidebar Toggles**:  
  * 「Draftを除く」「自分が作成したもののみ」などのクイックフィルタ。  
  * PR個別チェックボックスによる表示/非表示の切り替え。

## **3\. デスクトップ統合機能**

* **ブラウザ連携**: Desktop.getDesktop().browse(uri) を使用。  
* **OAuth連携**: 認証時に一時的なポートをリッスンし、認可後のリダイレクトを処理。  
* **グローバルショートカット**: Cmd \+ R (更新), Cmd \+ F (検索)。

## **4\. パフォーマンス最適化 (Rendering)**

* **Canvas Draw Optimization**: 画面外ノードの描画スキップ。  
* **LOD (Level of Detail)**: ズームアウト時の描画簡略化。  
* **非同期処理**: レイアウト計算や通信を Dispatchers.Default/IO で実行。

## **5\. UI状態の永続化**

* **Workspaceセッション**: 最後に見ていたリポジトリ、ズームレベル、トグル状態を保存。  
* **認証情報**: 取得したアクセストークンを OS のセキュアストレージ（macOS Keychain / Windows Credential Locker）に安全に保存。
