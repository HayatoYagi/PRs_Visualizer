# **GitHub PRs Visualizer 開発仕様書**

## **1\. プロジェクト概要**

GitHubのリポジトリにおけるOpen PRの状態を、WinDirStatのようなTreemap形式で可視化するデスクトップアプリケーション。複数のPR（特にAIエージェントによる並列作業）がどの範囲に影響を与えているかを直感的に把握し、コンフリクトの予兆や作業範囲の重複を監視することを目的とする。

## **2\. コア機能**

### **2.1. Codebase Visualization (Main View)**

* **Treemap描画**:  
  * 矩形の面積は、ファイルの総行数（LOC）またはディレクトリ内のファイル合計数に比例。  
  * ディレクトリ階層をネストして表示。  
* **ズーム・ナビゲーション**:  
  * マウスホイールによるスムーズな拡大・縮小。  
  * 拡大に伴い、深い階層のディレクトリ名やファイル名をフェードイン表示。  
* **PR情報のオーバーレイ**:  
  * 後述するカラーリング・ロジックに基づき、PRの作業範囲を可視化。

### **2.2. PR List View (Sidebar)**

* **PR一覧表示**: 現在リポジトリにあるOpen PRをリスト表示。  
* **フィルタリング**:  
  * Draft PRの表示/非表示切り替えトグル。  
  * 各PRごとの表示/非表示を切り替えるチェックボックス。  
* **詳細確認**: PRアイテムをクリックすると、ブラウザで該当するGitHub PRページを開く。

## **3\. 視覚的表現（ビジュアライゼーション・ロジック）**

情報の競合を避けるため、以下の「レイヤー構造」で描画を行う。

| レイヤー | 要素 | ロジック |
| :---- | :---- | :---- |
| **ベース** | 矩形の面積 | ファイルサイズ（行数等）に応じた分割。 |
| **作業主体** | **枠線の色 (Stroke)** | PRごと、またはAuthorごとに固有の色を割り当て。 |
| **変更種別** | **塗りつぶし色 (Fill)** | **緑系:** 追加(Addition) / **黄系:** 編集(Modification) / **赤系:** 削除(Deletion) |
| **変更の密度** | **彩度 / 透過度** | 「変更行数 ÷ ファイル総行数」の割合が高いほど、色を濃く（不透明に）表示。 |
| **衝突注意** | **ハッチング (斜線)** | 複数のPRが同じファイルを変更している場合、矩形に斜線パターンを重ねる。 |

## **4\. 技術スタック案（Kotlin Multiplatform）**

デスクトップアプリとして高いパフォーマンスと型安全性を両立する構成。

* **言語**: Kotlin  
* **UI Framework**: Compose Multiplatform (Compose for Desktop)  
* **Rendering Engine**: Skia (Compose標準。大量の矩形やアニメーション描画に最適)  
* **Networking**: Ktor (GitHub API v3/v4 通信)  
* **Auth**: OAuth2 (GitHub App または Personal Access Token)  
* **Local Database**: Exposed または SQLDelight (リポジトリ構造のキャッシュ用)  
* **Architecture**: MVVM (Model-View-ViewModel)

## **5\. データモデルとロジック**

### **5.1. リポジトリツリーの構築**

1. GitHub APIを使用して、対象ブランチ（デフォルトはmain）のファイルツリーを取得。  
2. 各ファイルの行数を取得し、Treemapアルゴリズム（Squarified Treemap等）で矩形座標を計算。

### **5.2. PR差分データのマッピング**

1. Open PRの files changed リストを取得。  
2. 各ファイルの additions / deletions 数値を解析。  
3. 同一ファイルパスに対して複数のPRが存在するかをチェックし、コンフリクトフラグを立てる。

## **6\. 将来的な検討事項 (v2+)**

* **近接コンフリクト検知**: 行レベルでの重複（diffのpatch解析）による、より精細な衝突警告。  
* **時系列再生**: PRが作成された順番に可視化を再生し、開発の流れを追体験する機能。  
* **AI連携**: コンフリクトが予想される場合に、解決案を提案するAIアシスタント機能。
